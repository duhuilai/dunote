# v0.1.35

## 图片持久化修复（关键）
- **切换文档后图片仍丢失（彻底修复）**：v0.1.34 的 `flush` 机制仍未覆盖两个真实场景：
  1. `sync effect` 误用 `note.content === lastEmittedRef.current` 判断，当切换到内容恰好相同的笔记（如空笔记粘贴图片后切到另一篇空笔记）会直接跳过 flush，未保存的编辑（含图片）随之丢失。现改为用 `noteId` 比较——**只要切换的是不同笔记，必定先 `flush` 旧笔记再加载新笔记**。
  2. 本地文件笔记（`_isLocalFile`）的编辑只写磁盘、从不更新内存里的 `localNotes` 数组，而 `selectedNote` 对本地文件取自 `localNotes` 旧值，导致切回时编辑器加载的是旧内容、图片消失。现 `flush` 本地文件笔记时**同时把最新内容同步回内存 `localNotes`**（新增 `onLocalPersist` 回调），并保留写磁盘作为持久化。

# v0.1.34

## 导出修复
- **图片粘贴后切换文档丢失**：根因是自动保存有 1.5s 防抖，快速切到其它文档时待保存内容（如刚粘贴的图片）会被新文档的 `setContent` 覆盖而从未落盘；且 `onUpdate` 闭包里的 `note.id` 在编辑器复用时可能失效写错笔记。现改为用 `editorMetaRef` 记录“当前编辑器加载的笔记”，切文档/卸载前先 `flush` 未落盘内容到正确的笔记，粘贴/拖入图片期间若已切换文档则丢弃本次插入避免污染其它笔记。
- **导出 Word 报错**：颜色来自 TipTap 的 Color 扩展时写为 `rgb(0, 0, 0)` 等格式，docx 要求 6 位十六进制导致 `Invalid hex value` 崩溃。现 `normColor` 支持 `rgb()/rgba()/#rgb/#rrggbb` 等统一转 6 位十六进制。
- **导出 PDF 空白**：离屏渲染容器原来放在 `left:-10000px`，html2canvas 从该屏幕外坐标截取导致整页空白。改为用 0 尺寸裁切外层把内容挡在视线外、被截元素本身位于视口坐标 (0,0)，确保内容被正确渲染。
- **导出 Markdown 表格丢失**：`turndown-plugin-gfm` 仅转换“首行全为 `<th>`”的表格，首行混合 `th/td` 的键值表会被原样保留为原始 HTML。现导出前先用 DOM 把任意 `<table>` 转为 GitHub 风格 Markdown 表格文本，保证表格内容不丢。

# v0.1.33

## 导出修复
- **导出 PDF 没反应**：原实现用 `window.print()` 走系统打印对话框，在 macOS WebView 上会静默无响应。改为前端用 `html2pdf.js`（html2canvas + jsPDF）直接生成真正的 `.pdf` 文件，跨平台稳定可用。
- **导出 Word 点了保存没反应**：根因是异常被静默吞掉、且 macOS 写入用户所选路径可能受安全限制。新增应用内自定义中文「保存」弹窗（文件名输入 + 中文「保存」按钮，写入「下载」文件夹），并对所有导出路径加 `try/catch` + toast 提示，成功/失败都可见；Word 生成改用 `Packer.toArrayBuffer` 更稳健。
- **导出保存按钮中文化**：原生系统保存对话框的确认按钮由操作系统绘制、无法汉化，故改为应用内自定义「保存」弹窗（按钮中文「保存」），并保留「选择其他位置…」走原生对话框。导出格式标签汉化为 Markdown 文档 / HTML 网页 / Word 文档 / PDF 文档。

## 智能表格
- **单选列**：空值选项不再显示「（空）」，改为空白。
- **日期列**：用自定义中文日历替换原生 `input[type=date]`，月份与星期均为中文（存储值仍为 ISO `YYYY-MM-DD`）。
