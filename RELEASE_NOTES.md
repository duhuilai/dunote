# v0.1.32

## 修复
- **编辑器图片/链接按钮没反应**：Tauri 的 WebView 默认不实现 `window.prompt`（mac 上直接返回 null，等于“点了没反应”），且 @tauri-apps/plugin-dialog v2 已移除 `prompt` API。新增 `src/utils/prompt.ts` 的自包含输入框（DOM 弹层，返回 Promise），工具栏「图片/链接」按钮、斜杠命令「图片」均改用它，现在能正常弹出输入。
- **智能表格「删除」按钮没反应**：同样因 `window.confirm` 不可用，改为 plugin-dialog 的 `confirm`。
- **智能表格表头「…设置」按钮点了没反应**：根因是表头单元格 `th` 设了 `overflow:hidden`，把列设置/选项编辑弹层裁掉了。改为 `th` 不再裁剪，仅在列名内层 div 裁剪文字，设置弹层正常显示。
- **行号被删除按钮覆盖**：行尾操作格里「行号」与「删除」按钮原本都在流内占位，36px 窄格放不下互相重叠。改为删除按钮绝对定位居中、默认隐藏，悬停时行号隐藏、删除按钮出现，互不遮挡。
- **排序/筛选交互异常**：原「点击外部关闭」逻辑会在点工具栏「排序/筛选」按钮的 mousedown 阶段把弹层先关掉、又被 click 重开，导致无法关闭/卡顿。改为排序筛选弹层只由自身 Popover 处理外部点击关闭。
- **粘贴/拖入的图片仍以 blob: 临时地址存储**：编辑器新增 `handlePaste` 与 `handleDrop`，把剪贴板/拖入的图片文件读取为 base64 dataURL 后内嵌插入，不再生成 `blob:tauri://localhost/...` 这类会随重启/更新丢失的临时地址。
