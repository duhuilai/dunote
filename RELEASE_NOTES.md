# v0.1.39

## 修复：图片切换/重启后丢失（真正的根因，关键修复）
- **根因**：编辑器的图片扩展未开启 `allowBase64`，TipTap 默认会在**从 HTML 解析内容**（切换/重启后 `setContent` 重新加载磁盘内容）时**丢弃 `data:image/...;base64` 的图片**。首次用命令插入图片时不受此限制，所以显示正常、也正确写入了磁盘（磁盘文件里确实有完整的 base64 `<img>`），但只要重新加载就被过滤掉——表现为文本正常、只有图片丢失。这与图片大小无关。
- **修复**：图片扩展改为 `Image.configure({ allowBase64: true })`，解析 HTML 时保留 base64 图片。
- **附带好处**：此前因该问题「看起来丢失」但实际仍存于磁盘的图片，升级后重新打开笔记即可**自动恢复显示**，无需手动处理。

## 修复：恢复历史后页面不刷新（需切换笔记才显示）
- **根因**：恢复历史时笔记 id 不变，编辑器的加载逻辑判定为「同一篇已加载」直接跳过，不重新载入内容，必须切换到别的笔记再切回来才刷新。
- **修复**：新增「强制重载」信号（`reloadToken`）——恢复历史后立即让当前笔记以磁盘/内存最新内容重新加载，页面即时刷新，无需再切换笔记。本地文件笔记在写盘成功后才触发重载，避免读到旧内容。

## 优化：任务负责人 / 参与人改为从人员中选择
- 新建任务时，「负责人」改为**下拉选择**已录入的人员，「参与人」改为**多选**（可勾选多名人员），不再手动输入。
- 便于后续按人员统计每个人参与 / 负责了哪些任务（存储与人员姓名对应）。
- 若尚未录入人员，会提示先到「人员管理」中添加。
